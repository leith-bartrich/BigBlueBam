# Railway environment variables

Auto-generated by `scripts/gen-railway-configs.mjs` from the service
catalog at `scripts/deploy/shared/services.mjs` and the env-var hints
at `scripts/deploy/shared/env-hints.mjs`. Re-run the generator whenever
either of those changes.

## Conventions on Railway

- **Internal DNS**: Railway services in the same project resolve each
  other as `<service-name>.railway.internal`. The hostnames in the
  catalog (`api`, `banter-api`, `mcp-server`, `minio`, `qdrant`, ŌĆ”)
  match the names you give the services in the Railway dashboard.
- **Plugin references**: managed plugins (Postgres, Redis) expose
  their connection strings as injectable variables you reference like
  `${{Postgres.DATABASE_URL}}` and `${{Redis.REDIS_URL}}`.
- **Cross-service references**: any variable on another service can be
  pulled in with `${{<service>.<VAR>}}` ŌĆö e.g. app services pull MinIO
  credentials from the minio service via `${{minio.MINIO_ROOT_USER}}`.
- **Public URL**: every variable below tagged "public" should be set to
  the public URL of the **frontend** service (the only externally-
  exposed service in this project).

## Secrets you generate yourself

Run these once and stash the values somewhere safe ŌĆö every service
that uses the same secret must get the SAME value or sessions and
inter-service auth will break.

```sh
# Shared session secret (api, helpdesk-api, banter-api, beacon-api, ŌĆ”)
openssl rand -hex 32

# api Ōåö helpdesk-api shared secret
openssl rand -hex 16

# Internal service-to-service secret (bolt-api event ingestion)
openssl rand -hex 16

# MinIO root credentials
openssl rand -hex 16   # MINIO_ROOT_USER
openssl rand -hex 24   # MINIO_ROOT_PASSWORD

# LiveKit API credentials (only if you use voice/video)
openssl rand -hex 16   # LIVEKIT_API_KEY
openssl rand -hex 32   # LIVEKIT_API_SECRET
```

Each table lists every env var for that service. The **Value** column
shows what to type into the Railway dashboard, with `<placeholders>`
for things you have to fill in yourself. Required variables are flagged
with **R**, optional with `o`.
